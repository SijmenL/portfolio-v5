de Sleutelzoeker

23-09-2022

Hogeschool Rotterdam

Sijmen, Viggo, Timon, Michiel

Laatst bewerkt op: 06-11-2022

De Sleutelzoeker is een robotje dat op een willekeurige wijze op de grond rond rijd, totdat het tegen een metalen object (zoals bijvoorbeeld je sleutels) aan botst en aangeeft dat hij iets gevonden heeft. Ik heb de code gemaakt in JavaScript en het robotje zelf gebouwd.

Communicatie

Ik heb samen met mijn peerteamgenoot een klein interview afgenomen over de dagelijkse dingen die niet fijn zijn. Daaruit bleek dus dat hij wel eens zijn sleutels kwijt raakt, waar ik mijn robot verder op heb gebaseerd. Hij is dus ook gelijk de doelgroep waarop ik het project heb afgestemt. Voor de robot heb ik mijn vorige pushy robotje opnieuw ontworpen.

Ik heb een powerpoint gemaakt om wat duidelijker te laten zien wat de bedoeling was van de robot. Verder laat ik zien wat de gedachtegang en het idee achter de robot is. Ik heb de presentatie afgestemd op het publiek, dus ik heb meer visueel laten zien dan met tekst, daarom denk ik dat ik in ontwikkeling ben voor dit onderdeel.

Ondernemen

Ik heb de programmering van de robot volledig gemaakt en heb ik ook het bakje een paar keer opnieuw moeten maken. Ik heb alleen aan deze robot gewerkt. Ook heb ik meerdere problemen die de robot had opgelost, niet alle problemen kon ik eruit halen, maar dat komt vooral omdat de touch sensor van de circuit playground niet meer betrouwbaar werkt als het niet zit ingeplugd in de computer. Ik vind dat ik nog zeker een beginner ben op dit gebied, aangezien er nog veel is wat ik nog niet weet.

Leren en reflectief vermogen

Ik heb op de eerste versie van mijn ontwerp feedback gevraagd aan mijn peerteamgenoot. Ik heb het ontwerp en de werking van de robot aangepast op de dingen die we toen besproken hebben, zoals dat de robot achteruit zou moeten kunnen rijden als hij vast zit. In de toekomst zou ik tussentijds wat meer feedback moeten vragen, omdat dit waardevolle informatie zou kunnen zijn voor het verdere ontwerp proces. Ik denk dat ik in ontwikkeling ben, maar in de toekomst wel naar op niveau zou kunnen gaan.

  • Tip: Betere looks, zou er veel leuker uitzien
  • Top: Leuke functie, random rondrijden.
  • Tip: Probeer het probleem dat je robot de sleutels niet kan vinden op te lossen of een ander doel voor de robot
  • Top: Grappige en duidelijke uitleg
  • Tip: Misschien als iets niet helemaal lukt ben je iets te hoog gegrepen kijk misschien of er nog andere manieren om dit probleem op telossen
  • Top: De uitwerking is goed ook al deed hij het niet helemaal
